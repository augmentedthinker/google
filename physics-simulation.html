Physics Simulation

A simple gravitational simulation where objects interact within a constrained space. These bouncing spheres demonstrate how fundamental forces like gravity and friction can create realistic, organic motion in a digital environment.

Technical Notes

This artifact uses a basic 2D physics engine built from scratch. Each sphere is an object with properties for position, velocity, and radius. By updating these properties every frame, we simulate motion within a coordinate system.

Force Vectors

On every animation frame, a "gravity" constant is added to the vertical velocity of each sphere. This causes them to accelerate downward. When a sphere hits the bottom boundary, we invert its velocity (making it bounce) and apply a "friction" multiplier to simulate energy loss over time.

Collision Detection

The simulation calculates the distance between the center of each sphere and the boundaries of the canvas. If that distance plus the radius exceeds the canvas edge, a collision is triggered, reversing the velocity vector.

Integration Loop

The simulation uses Euler integration, where velocity is updated by acceleration, and position is then updated by velocity. It is a straightforward approach that remains computationally light, allowing for smooth motion even on mobile devices.
