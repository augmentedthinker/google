## Current Protocol Notes

### Startup

Startup should remain streamlined. The agent should not automatically read memory files on every boot. Christopher can explicitly ask for memory review when he wants it.

### Memory Push

A memory push in the active repo means:

1. Create a detailed local memory under the shared workspace `memory/` directory.
2. Mirror that memory into the active repo’s memory archive.
3. Include date, time, active model ID, repo context, narrative summary, decisions, file changes, and continuity notes.
4. Add a newest-first card to the repo’s `memories.html`.
5. Keep the local memory and repo mirror aligned in substance.

### Artifacts

A new artifact should usually include:

1. A standalone artifact page under `artifacts/`.
2. A newest-first card on `artifacts.html`.
3. A title and signature with date, time, and model ID.
4. The universal nav unless the artifact intentionally requires a full immersive presentation.

### House Style

Shared styles now belong in `assets/css/site.css`. New pages should use the shared shell, hero, card, typography, and nav patterns unless Christopher asks for a different direction. `style-guide.html` should be updated whenever we add reusable site conventions.

## Continuity Notes

The Codex repo has now crossed from initial prototype into a maintainable static site architecture. The most important future maintenance principle is to avoid copying large inline CSS into every new page. Use the shared stylesheet first, keep page-specific styles local only when necessary, and update the House Style page when a pattern becomes reusable.

Christopher is actively shaping this repo as the Codex-specific counterpart to the Google repo, but with its own visual identity and Codex-oriented architecture. The site should remain clean, internal-navigation-first, and easy for future model sessions to understand quickly.
